## The big re-scoping calls (feasibility-verified — these DROP/DEFER audit items)

- **DROP `torchaudio.forced_align`** (audit item 19): no torchaudio wheel matches the installed
  torch 2.12.1 (latest is 2.11.0); installing force-downgrades torch and risks breaking
  sentence-transformers 3.4.1 / TreeSeg (the structure stage). It also fixes the WRONG error — both
  audits show `ends_on_period_rate = 1.0` yet clips still end mid-clause, so the dominant residual
  is **segmentation (wrong word), not ~90ms timing**. BND1 (free text guards) + BND2 (re-time the
  LLM boundary with the already-installed faster-whisper) cover the real error.
- **DROP `inaSpeechSegmenter`** (audit item 22): pulls unpinned TensorFlow (~1GB) + deprecated
  Pyro4 into the shared torch/numpy venv; tensorflow-metal has no py3.12 wheel (CPU-only anyway).
  **Never add TF/Pyro4 to clips/.venv.** One Gemini-audio classify call (GEN4) gives equivalent
  music-vs-speech/singing discrimination at zero venv risk.
- **DEFER SaT/wtpsplit** (audit item 17): largely redundant with the LLM punctuation-restoration
  package that already produces terminators for the default supadata source. It only helps the rare
  `sentences_from_chunks` fallback — whose highest-value fix (killing fabricated terminators) is
  free in BND1.
- **DEFER parakeet default-flip**: adopt opt-in / auto-trigger only (ASR1). parakeet-mlx 0.5.2
  installs clean (pure MLX, no torch dep) but flipping the default needs a golden-set A/B that
  doesn't exist yet, and auto-trigger forces an audio download that negates supadata's latency edge.
- **DEFER diarization-based interview/story genre tier**: pyannote not installed, gated on HF token;
  low ROI now.
