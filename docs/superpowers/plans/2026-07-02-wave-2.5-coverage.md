## Root causes this wave fixes (all verified with file:line, most replayed deterministically)

1. **Cross-indexer cache poisoning** (THE "stale structure" root cause, sharpened): run_eval
   builds sentences with legacy `build_sentence_index` (run_eval.py:46-52) and unconditionally
   `save_structure`s into the shared app cache (run_eval.py:178); the app builds
   punctuation-restored sentences (orchestrator.py:105-106). structure.json has NO fingerprint
   beyond `schema_version` (models.py:197-207). Out-of-range sentence indices are silently
   clamped to the last sentence at assembly (candidates.py:428-431). qP's current cache indexes
   sentences to 321 against the app's 183 — poisoned for the app RIGHT NOW.
2. **Punctuation cache persists degraded results forever**: `status='degraded'` chunk artifacts
   (30-37s run-on "sentences", qP tail) are cached and never retried
   (punctuation/cache.py:44-59 checks fingerprint/model/prompt_version but NOT status) — the
   upstream feeder of unit-boundary drift.
3. **Unit-boundary drift is silent**: the `s0 = max(s0, cursor)` clamp cascade (units.py:165-190)
   shifts every subsequent unit when the LLM double-claims a run-on sentence; `Unit.warnings`
   exists but is never written; units are not a partition (23 uncovered sentence indices on qP).
4. **Forward-in-time requires edges**: `_introducer` returns `lst[0]` — a FUTURE unit — when no
   prior introducer exists (dependencies.py:42-43). Post-Q2 nutrition this multiplied: qP's
   vector/scalar defs (108-115s) now "require" the magnitude def at 232.9s. Pollutes closure
   referential lists, cards, judge prerequisite inputs. LLM edge-pass failure is also fully
   silent (dependencies.py:199-201, bare except, no degraded note).
5. **Closure is granularity-dependent** (the items-3/4 regression): 'after'-position contract
   elements take only `cand[0]` (closure.py:76-78), so the rebuilt fine-grained structure
   (139→239 units) yields a 24.2s vectors/scalars sliver covering item 3 at 0.548 (<0.60 rule)
   and item 4 at 0.000 — pre-rebuild the same anchor spanned 92.72-189.65 covering both at 1.0.
6. **Coverage floor counts "has an anchor", not "content covered"** (candidates.py:330-348):
   one 3.3s anchor marks an 81s topic covered; chapter_coverage has the identical blind spot.
7. **Global PLAN_ROLE_CAP starves claim-dense zones** (items 21-22): 46 eligible claims compete
   for 4 video-wide slots (config.py:238, candidates.py:312-328); role-capped max non-arc
   anchors ≈ 35 vs budget 32 — saturation, not budget, is now the binding constraint.
8. **Arc terminal blindness + micro-arc noise** (item 16): TERMINAL_ROLES={result,solution} +
   calculation-as-final only (arcs.py:32-35, 98-104). The 205s graph-build example (16
   worked_steps, zero result/solution) is invisible; meanwhile 8 of qP's 11 detected arcs are
   2-15s Socratic micro-pairs (two inside the sponsor promo), all saturation-exempt and
   snap-padded to 20s. No substance floor exists anywhere.
9. **Trim routing is unreachable via the prompt** (n_trims=0 root cause): JUDGE_SYSTEM
   (validate.py:70-106) never asks about excess/over-inclusion, so verdicts never carry the
   kinds `_trim_flavored` (validate.py:489-506) requires. The trim bisection is built and tested
   — it's missing its input. Atomicity reasons are quote-friendly (the second idea's text is
   verbatim in the span), so they can survive the asymmetric kill gate, unlike absence kinds.
10. **Same-facet dedupe manufactures mashes** (dominant deterministic arc killer): every
    worked-example-family role shares facet 'worked_example' (base.py:70-74), so distinct
    problems that touch get union-merged (boundary_adapt.py:140-148) — qP arcs 2+4+5 (three
    distinct practice questions) union into one [382,480] spec; unions have NO max-duration
    re-check; `merge_partb` gives all non-span keys to the winner so arc_id is silently
    stripped (integrity.py:51,58) and n_arc_clips undercounts.
11. **Refund lock-in traps**: any 1-sentence overlap with a shipped spec kills a refund
    candidate outright ('refund overlap loser', __init__.py:326-337) — a superset that would
    fix sliver coverage is dropped; re-built killed spans hit the (unit_ids, text_hash) verdict
    cache and silently re-die. Round-0 mistakes are permanent.
12. **Severed-pair linker fires 0 by construction**: closure force-inlines the required
    problem_statement into every payoff clip (fails "later has no setup"); opener roles are
    non-anchor so setup-only clips barely exist; SEVERED_OPENER_ROLES omits practice_prompt —
    the most common opener (sequence.py:19, :91-103); adjacency-only detection.
13. **Repair converts practice clips into solution clips**: expansion keys off verdict booleans
    regardless of the bound contract — `pull_result()` fires on a practice_prompt clip whose
    contract deliberately excludes the answer (validate.py:402-403, base.py:114-118); the
    specificity tie-break then makes the conversion permanent (contracts.py:63).
14. **Eval blind spots**: no inventory-recall metric (qP has NO golden file at all — topic
    defaults to "" and every gold metric silently skips the coverage flagship); chapter_coverage
    is self-referential (structure grades itself); shipped spans/plan/ledger are never persisted
    (jobs are in-memory; this diagnosis required offline reconstruction); quality_floor=0.45 is
    an inline literal (__init__.py:356); phantom_verdict_rate mixes structurally-unquotable
    absence kinds with genuinely-quotable ones and mostly measures the 4b post-snap pass.
